# PLC 제어 코드 초안 생성 자동화 포트폴리오

자연어로 작성된 공정 설명과 IO 정보를 바탕으로 PLC 제어 코드 초안과 검토용 래더 문서를 생성하는 개인 포트폴리오 프로젝트입니다.

이 프로젝트는 PLC 엔지니어의 업무를 대체하기 위한 것이 아니라, 반복적인 초안 작성, IO 정리, 제어 조건 문서화 작업을 보조하는 도구를 목표로 합니다.

---

## 프로젝트 개요

제조 현장에서는 공정 사양서, IO 목록, 장비 동작 조건을 바탕으로 PLC 제어 프로그램을 작성합니다. 이 과정은 반복 작업이 많고, 초기 초안 작성에도 많은 시간이 소요됩니다.

본 프로젝트는 이러한 과정을 자동화하기 위해 만들었습니다.

입력으로는 사람이 작성한 공정 설명을 사용하고, 출력으로는 다음과 같은 결과물을 생성합니다.

- PLC ST 코드 초안
- LS/XGT 기준 IO 주소 매핑표
- XGT 검토용 Ladder Draft
- 자동 검증 리포트

---

## 현재 구현된 기능

현재 Batch 공정 샘플을 기준으로 다음 기능을 구현했습니다.

- 자연어 공정 설명 입력
- IO 신호 자동 추출
- 입력, 출력, 내부 변수 구분
- 공정 상태 흐름 구성
- 인터락 조건 정리
- 알람 조건 정리
- LS/XGT용 ST 코드 초안 생성
- X/Y/M/D 주소 매핑 생성
- XGT 검토용 Ladder Draft 생성
- 생성 결과 자동 점검 리포트 생성

## 샘플 공정

현재 테스트에 사용한 샘플은 3단 가열 혼합 Batch 공정입니다.

공정 흐름은 다음과 같습니다.

원료 투입  
↓  
가열  
↓  
교반  
↓  
배출  
↓  
세척

포함된 주요 조건은 다음과 같습니다.

- 자동 운전 조건
- 수동 운전 조건
- 비상정지 조건
- 수위 조건
- 과열 방지 조건
- 배출 인터락
- 타임아웃 알람
- 알람 래치 및 리셋

## 생성 결과 예시

테스트 실행 후 다음과 같은 파일이 생성됩니다.

outputs/  
├─ 날짜_ls.st  
├─ 날짜_ls_mapping.csv  
├─ 날짜_ls_ladder.txt  
├─ 날짜_ls_ladder_mapping.csv  
└─ xgt_flow_report_날짜.txt

각 파일의 의미는 다음과 같습니다.

| 파일 | 설명 |
|---|---|
| 날짜_ls.st | LS/XGT용 ST 코드 초안 |
| 날짜_ls_mapping.csv | IO 신호와 X/Y/M/D 주소 매핑표 |
| 날짜_ls_ladder.txt | XGT에서 검토할 수 있는 Ladder Draft |
| 날짜_ls_ladder_mapping.csv | Ladder Draft 기준 주소 매핑표 |
| xgt_flow_report_날짜.txt | 생성 결과 자동 점검 리포트 |

## 최근 테스트 결과

최근 테스트에서는 다음 항목을 확인했습니다.

[OK] LS/XGT ST 파일 생성  
[OK] LS/XGT Ladder Draft 파일 생성  
[OK] LS/XGT Mapping CSV 생성  
[OK] BATCH OUTPUT 포함  
[OK] 주요 출력 신호 포함  
[OK] Ladder Network 생성  
[OK] 출력 로직 자동 반영

생성된 Ladder Draft에는 다음과 같은 구성이 포함됩니다.

- Safety 네트워크
- State Transition 네트워크
- Interlock 네트워크
- Alarm 네트워크
- Derived 조건 네트워크
- Batch Output 네트워크

## 프로젝트에서 중점적으로 본 부분

이 프로젝트에서 가장 중요하게 본 부분은 단순히 코드를 생성하는 것이 아니라, 현장 엔지니어가 검토할 수 있는 형태로 결과물을 정리하는 것입니다.

그래서 다음 항목을 특히 신경 썼습니다.

- 사람이 읽을 수 있는 출력 구조
- IO 주소 매핑표 제공
- 안전 조건과 알람 조건 분리
- 상태 흐름을 단계별로 확인 가능하게 구성
- 자동 테스트 리포트 생성
- 생성 결과를 제출용 폴더로 정리

---

## 실행 예시

테스트용 샘플을 실행하려면 다음 명령어를 사용합니다.

python test_xgt_flow.py

LS/XGT용 ST와 Ladder Draft를 직접 생성하려면 다음 명령어를 사용할 수 있습니다.

python cli.py --text-file xgt_sample_spec_batch.txt --recipe batch_control --vendor ls --format both

테스트가 끝난 후 제출용 폴더를 만들려면 다음 명령어를 사용합니다.

python make_xgt_submission_pack.py

## 현재 상태

현재 단계는 다음과 같습니다.

자연어 수주서  
↓  
IO 추출  
↓  
제어 흐름 구성  
↓  
LS/XGT ST 생성  
↓  
Ladder Draft 생성  
↓  
자동 검증 리포트 생성

위 흐름까지 동작하는 것을 확인했습니다.

현재 Ladder Draft는 XG5000 자동 import 파일이 아니라, XGT에서 사람이 네트워크 단위로 검토하고 입력할 수 있는 문서 형태입니다.

## 포트폴리오 관점에서의 의미

이 프로젝트는 제가 다음 역량을 갖추고 있음을 보여주기 위한 포트폴리오입니다.

- 공정 요구사항을 구조화하는 능력
- IO 목록과 제어 조건을 정리하는 능력

- 반복 작업을 자동화하는 문제 해결 능력

---

## 사용 전제

본 프로젝트의 결과물은 PLC 제어 코드 초안입니다.

현장 장비에 직접 적용하기 전에는 반드시 PLC 엔지니어의 검토와 테스트가 필요합니다. 이 프로젝트는 실무자의 검토 시간을 줄이고, 초기 초안 작성과 문서화 과정을 보조하는 것을 목표로 합니다.
